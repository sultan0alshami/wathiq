# حساب ضابط الرحلات (هاني بخش)

> يستخدم هذا الحساب للوصول إلى قسم **الرحلات** فقط (دون بقية الوحدات المالية/الإدارية).

| الحقل | القيمة |
| --- | --- |
| الاسم | هاني بخش |
| البريد الإلكتروني المقترح | `trips@wathiq.com` |
| كلمة المرور الأولية | `Wathiq@Trips2024` *(قم بتحديثها بعد أول تسجيل دخول)* |
| الدور في Supabase | `trips` |
| مسار الهبوط عند الدخول | `/manager/trips` |

## خطوات الإنشاء في Supabase

1. من لوحة Supabase → Authentication → Users → Add user.
2. أدخل البريد الإلكتروني وكلمة المرور أعلاه (أو استخدم كلمة مرور آمنة مختلفة).
3. بعد إنشاء المستخدم انسخ الـ UUID الخاص به ثم نفّذ الاستعلام التالي في تبويب SQL:

```sql
insert into public.user_roles (user_id, role, name)
values ('<USER_UUID>', 'trips', 'هاني بخش')
on conflict (user_id) do update set role = excluded.role, name = excluded.name;
```

> بمجرد حفظ السجل سيتم التعرف على الدور بشكل تلقائي داخل الواجهة، وسيظهر قسم الرحلات فقط لهذا المستخدم.

## تنبيهات أمنية

- يُنصح بتفعيل **تغيير كلمة المرور الإجباري** عند أول تسجيل دخول.
- هذا الحساب يملك صلاحية رفع الصور والتقارير عبر API `/api/trips/sync`؛ تأكد من حماية الأجهزة المستخدمة.
- إذا تم فقدان الجهاز أو الشريحة قم بإلغاء الجلسات المفتوحة من لوحة Supabase.
